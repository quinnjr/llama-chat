# Contributing to llama-chat

Thanks for your interest in contributing. This document covers the rules and workflow for the project.

## Getting Started

1. Fork the repo and clone it
2. Create a feature branch from `develop`: `git checkout -b feature/your-feature develop`
3. Make your changes, write tests, run `cargo test` and `cargo clippy`
4. Push and open a PR against `develop`

## Branching Model

This project uses git-flow:

- **`main`** — stable releases only. Never commit directly.
- **`develop`** — integration branch. Feature branches merge here.
- **`feature/*`** — branch from `develop`, merge back to `develop`.
- **`hotfix/*`** — branch from `main`, merge to both `main` and `develop`.

## Commit Messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
<type>(<scope>): <subject>

<body>
```

Types: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert`

The subject line should be imperative, lowercase, under 72 characters, no trailing period. The body should explain **why**, not what — the diff shows what changed.

## Code Standards

- Rust edition 2024, minimum Rust 1.85
- `cargo clippy` must pass with zero warnings
- `cargo test` must pass — aim for high coverage
- No `unsafe` without justification and a safety comment
- Follow existing patterns in the codebase

## AI-Assisted Code Attribution

**If you use AI coding assistants (Claude, Copilot, ChatGPT, Gemini, local models, or any other AI tool) to generate, modify, or substantially inform code in your contribution, you must disclose this in your pull request description.**

This is not optional. Attribution is required for transparency and intellectual honesty.

### What to disclose

Include a section in your PR description:

```markdown
## AI Attribution

- **Tool(s) used:** Claude Code, GitHub Copilot, etc.
- **Scope:** Describe what was AI-generated or AI-assisted (e.g., "entire implementation", "test generation", "refactoring suggestions", "boilerplate only")
- **Human review:** Confirm you reviewed, tested, and understand all AI-generated code
```

### What counts as AI-assisted

- Code generated by an AI tool, even if you edited it afterward
- Code where an AI suggested the approach or architecture you followed
- Tests, documentation, or config files generated by AI
- Using AI to translate code from another language

### What doesn't need attribution

- Using AI for research or learning that informed your approach (without copying output)
- Autocomplete suggestions of a few tokens (standard IDE behavior)
- Spell checking or grammar correction

### Why this matters

This project was built with AI assistance and we're transparent about that. We expect the same from contributors. AI tools are welcome here — hiding their use is not.

**Pull requests that appear to contain undisclosed AI-generated code may be rejected.**

## Pull Request Process

1. Ensure your branch is up to date with `develop`
2. All tests pass, clippy is clean
3. PR description explains what and why
4. Include AI attribution if applicable
5. One approval required to merge

## Reporting Issues

Open an issue with:
- What you expected
- What happened instead
- Steps to reproduce
- Your environment (OS, Rust version, Ollama version)
